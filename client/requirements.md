## Packages
zustand | Local state management with persistence for offline garden saving
uuid | Generating local unique IDs for garden boxes
framer-motion | Smooth animations for placing plants and page transitions
clsx | Class merging
tailwind-merge | Class merging

## Notes
The application relies heavily on local storage for saving garden layouts (Zustand persist).
Backend is primarily used as a read-only data catalog for plants and location data.
Grid coordinate system uses "x,y" string keys representing 1x1 foot squares.
